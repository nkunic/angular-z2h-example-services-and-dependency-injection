ANGULAR Services and Dependency Injection

Service

  • TS class used for sharing data or funcionality troughout the app
  • no need to specify any decorator as with components @Componennt
  • need to provide sevice either in the component or in the module
    • component: all child components get the service
    • in module: all components in module get the service
    • in root module: all app gets the service
  • Service can be injected into another service, and then neccesserey to add @Injectable decorator to the service which is receiving another service, but new angular rules state to add @Injectable to all services!

  • To provide it for the whole application either inside service

    @Injectable( providedIn: 'root')

    or in app.module.ts

    providers: [ShapesService]

Uses of services:

  • code reuse
  • cross-compomponent comunication
